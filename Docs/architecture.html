🏗️ System Architecture

JobHunter AI — Full Technical Design & Data Flow

📐 System Overview

JobHunter AI is a personal job hunting automation system. It scans the internet daily for job postings matching your profile and domain, builds tailored resumes using AI, and automatically submits applications. The system is domain-driven — everything is scoped to the active domain (University, Bank, Insurance, etc.).

🛠️ Technology Stack

  • Frontend: React 18 + Vite, React Router, Axios, TailwindCSS
  • Backend: Python 3.11+, FastAPI, SQLAlchemy ORM, Pydantic v2
  • Database: SQLite (via SQLAlchemy), Alembic for migrations
  • Scheduler: APScheduler — daily job scans at configurable time
  • Web Scraping: Playwright + BeautifulSoup4 for job board scraping
  • AI / LLM: OpenAI API (GPT-4) or local Ollama for resume tailoring
  • Document Export: python-docx (DOCX), WeasyPrint (PDF)
  • Email: smtplib / SendGrid for application submission
  • Auth: Single-user, local JWT token (no cloud auth needed)

🌐 Domain-Driven Flow

The Domain is the top-level context. Every module — job search, resume, institutions — is filtered by the active domain.

⚡ Key API Endpoints

# Domains
GET    /api/domains              # List all domains
POST   /api/domains              # Create domain
PUT    /api/domains/{id}/activate # Set active domain

# Jobs
GET    /api/jobs                 # List jobs (filtered by active domain)
GET    /api/jobs/{id}            # Job detail
POST   /api/jobs/scan            # Trigger manual scan
GET    /api/jobs/new             # Jobs found since last visit

# Institutions
GET    /api/institutions         # List institutions in active domain
POST   /api/institutions         # Add institution
PUT    /api/institutions/{id}/favorite  # Toggle favorite

# Resume
POST   /api/resumes/build        # Generate resume for a job
GET    /api/resumes/{id}/pdf     # Download PDF
GET    /api/resumes/{id}/docx    # Download DOCX

# Applications
GET    /api/applications         # List all applications
POST   /api/applications         # Submit application
PUT    /api/applications/{id}    # Update status/notes

# Profile
GET    /api/profile              # Get user profile
PUT    /api/profile              # Update profile
POST   /api/profile/skills       # Add skill
POST   /api/profile/experience   # Add experience
